> **Purpose:** Use this file as the instruction set when generating a new application.
> It captures the exact frontend stack, backend architecture, folder structure,
> database schema patterns, component conventions, and D365 integration approach
> used in a proven presales-demo application. Adapt the **domain entities** to
> your scenario; keep the **patterns and structure** identical.

## 7. Auth Pattern

**Strategy:** Simple email-only lookup against an `app_users` table. No passwords,
no Supabase Auth, no JWT. Session persisted in `localStorage`.

### AuthContext.jsx

```
Provider exports: { user, loading, error, isManager, isOperator, login, logout, switchUser }
```

- **login(email):** Query `app_users` where `email = input AND is_active = true`, `.single()`
- **Session persistence:** `localStorage.getItem/setItem('grower_user_email')`
- **On mount:** Check localStorage → auto-load user
- **Roles:** `manager` (full access) and `operator` (limited — no setup pages)
- **isManager:** `user?.role === 'manager'`

### Role Guarding

Two components:
1. **RequireRole** — Route-level guard. Wraps children; redirects to `/` via `<Navigate>` if role mismatch.
2. **RoleGuard** — Inline guard. Renders an "Access Restricted" message instead of redirecting.

Role enforcement is **client-side only**. Setup nav items are conditionally rendered
with `{isManager && ...}` in the sidebar.

## 11. React Query Hook Patterns

### File Organization
| File | Purpose | Hook Naming |
|---|---|---|
| `useSupabase.js` | Domain entity queries + mutations | `use{Entity}`, `useCreate{Entity}`, `useUpdate{Entity}Phase` |
| `useManagement.js` | CRUD for setup/admin pages | `use{Entities}`, `useCreate{Entity}`, `useUpdate{Entity}`, `useDelete{Entity}` |
| `useD365.js` | D365 integration operations | `useD365{Entities}`, `useExtractD365{Entities}`, `useUpsert...` |

### Query Key Conventions
```js
['fields']                    // Simple entity list
['grow-cycles', filters]      // List with filters object
['grow-cycle', id]            // Single record by ID
['dashboard-kpis']            // Aggregated dashboard data
['d365-products', filters]    // D365 cache queries
```

### Query Pattern
```js
export function useGrowCycles(filters = {}) {
  return useQuery({
    queryKey: ['grow-cycles', filters],
    queryFn: async () => {
      let query = supabase
        .from('grow_cycles')
        .select(`*, field:fields(*), crop:crops(*)`)
        .order('created_at', { ascending: false })

      if (filters.phase) query = query.eq('phase', filters.phase)
      if (filters.field_id) query = query.eq('field_id', filters.field_id)

      const { data, error } = await query
      if (error) throw error
      return data
    },
  })
}
```

- Relational data uses Supabase nested `.select()`: `*, field:fields(*), crop:crops(*)`
- Conditional filters applied via chained `.eq()` / `.gte()` / `.lte()`
- `enabled: !!id` for detail queries dependent on a route parameter

### Mutation Pattern
```js
export function useCreateGrowCycle() {
  const qc = useQueryClient()
  return useMutation({
    mutationFn: async (payload) => {
      const { data, error } = await supabase
        .from('grow_cycles')
        .insert(payload)
        .select(`*, field:fields(*), crop:crops(*)`)
        .single()
      if (error) throw error
      return data
    },
    onSuccess: () => {
      qc.invalidateQueries({ queryKey: ['grow-cycles'] })
      qc.invalidateQueries({ queryKey: ['dashboard-kpis'] })
    },
  })
}
```

- **Insert:** `.insert(payload).select().single()` — always return the created record
- **Update:** `.update({ ...updates, updated_at: new Date().toISOString() }).eq('id', id).select().single()`
- **Delete:** `.delete().eq('id', id)` — no return value, just invalidate
- **onSuccess:** Invalidate related query keys (including dashboard if counts change)
- **No optimistic updates.** No `onError` handlers. Errors propagate to React Query default handling.
- **Multi-step mutations:** Create parent → insert children in sequence (not parallel)

### Management CRUD Pattern (useManagement.js)
```js
// Full CRUD per entity — all follow this exact shape:
export function useCrops() { ... }           // useQuery, queryKey: ['crops']
export function useCreateCrop() { ... }      // useMutation, invalidates ['crops']
export function useUpdateCrop() { ... }      // useMutation, destructure { id, ...updates }
export function useDeleteCrop() { ... }      // useMutation, takes id directly
```

## 13. D365 Integration Architecture

### Data Flow
```
Browser (React)
  ↓ POST { action, params }
Supabase Edge Function (d365-proxy)
  ↓ OAuth client-credentials → Azure AD token
  ↓ OData GET/POST → D365 F&SCM
  ↓ Upsert results → Supabase tables (via service_role key)
  ↓ Return { count, upserted }
Browser reads cached data from Supabase tables
```

### d365Api.js Pattern (Frontend Service Layer)

```js
// Central proxy caller
async function callD365Proxy(action, params = {}) {
  let token = import.meta.env.VITE_SUPABASE_ANON_KEY
  try {
    const { data: { session } } = await supabase.auth.getSession()
    if (session?.access_token) token = session.access_token
  } catch { /* use anon key */ }

  const res = await fetch(`${EDGE_FN_BASE}/d365-proxy`, {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      'Authorization': `Bearer ${token}`,
      'apikey': import.meta.env.VITE_SUPABASE_ANON_KEY,
    },
    body: JSON.stringify({ action, ...params }),
  })
  // ... error handling
  return res.json()
}
```

Three categories of functions:
1. **Extract (inbound):** `extractD365Products()`, etc. → calls proxy → logs to `d365_sync_log`
2. **Push (outbound):** `pushYieldToD365()`, `pushHarvestToD365()` → proxy → D365 POST
3. **Read (cache):** `getD365Products(filters)` → direct Supabase `.from()` select on cache tables

## 14. Database Schema Patterns

### Table Conventions

| Convention | Pattern |
|---|---|
| **Primary key** | `id uuid primary key default uuid_generate_v4()` |
| **Timestamps** | `created_at timestamptz default now()`, `updated_at timestamptz default now()` |
| **Foreign keys** | `field_id uuid references fields(id) on delete cascade` |
| **Column names** | `snake_case` throughout |
| **Booleans** | `is_` prefix: `is_active`, `is_default`, `is_catch_weight` |
| **Status enums** | `text not null check (status in ('a','b','c'))` — NOT Postgres enums |
| **Generated columns** | `is_short_lead_time boolean generated always as (lead_time_hours <= 48) stored` |
| **Numerics** | `numeric(12,2)` for quantities/prices, `numeric(10,2)` for area |
| **JSON storage** | `raw_payload jsonb` on all external-system cache tables |
| **D365 natural keys** | `d365_` prefix: `d365_item_number`, `d365_warehouse_id` (unique) |
| **Composite unique** | `unique(harvest_schedule_id, team_member_id)` for junction tables |
| **Indexes** | Named `idx_{table}_{column}` on FK columns, status columns, date columns |

### Core Domain Tables Template

```sql
create extension if not exists "uuid-ossp";

-- Main entity
create table if not exists {entities} (
  id uuid primary key default uuid_generate_v4(),
  name text not null,
  status text not null default '{default}' check (status in ('{status_a}', '{status_b}', ...)),
  -- domain columns
  created_at timestamptz default now(),
  updated_at timestamptz default now()
);

-- Lifecycle tracking (links entity to details)
create table if not exists {entity}_cycles (
  id uuid primary key default uuid_generate_v4(),
  {entity}_id uuid references {entities}(id) on delete cascade,
  phase text not null default '{initial_phase}' check (phase in (...)),
  -- date columns for each phase transition
  created_at timestamptz default now(),
  updated_at timestamptz default now()
);

-- Scheduling
create table if not exists {entity}_schedules (
  id uuid primary key default uuid_generate_v4(),
  {entity}_cycle_id uuid references {entity}_cycles(id) on delete cascade,
  scheduled_date date not null,
  start_time time,
  end_time time,
  status text not null default 'scheduled' check (status in ('scheduled','in_progress','completed','cancelled')),
  notes text,
  created_at timestamptz default now(),
  updated_at timestamptz default now()
);

-- Schedule ↔ Team Member junction
create table if not exists {entity}_schedule_members (
  id uuid primary key default uuid_generate_v4(),
  {entity}_schedule_id uuid references {entity}_schedules(id) on delete cascade,
  team_member_id uuid references team_members(id) on delete cascade,
  unique({entity}_schedule_id, team_member_id)
);

-- Measurement / yield records
create table if not exists {entity}_records (
  id uuid primary key default uuid_generate_v4(),
  {entity}_cycle_id uuid references {entity}_cycles(id) on delete cascade,
  quantity numeric(12,2) not null,
  unit_of_measure text not null default 'lbs',
  grade text check (grade in ('A','B','C','reject')),
  recorded_at timestamptz default now(),
  notes text
);

-- D365 outbound sync queue
create table if not exists d365_sync_queue (
  id uuid primary key default uuid_generate_v4(),
  entity_type text not null,
  entity_id uuid not null,
  payload jsonb not null,
  status text not null default 'pending' check (status in ('pending','processing','synced','failed')),
  attempts integer default 0,
  last_error text,
  created_at timestamptz default now(),
  synced_at timestamptz
);
```

### RLS Strategy (Demo / Presales)

All tables use permissive policies for simplicity:
```sql
alter table {table} enable row level security;
create policy "read" on {table} for select using (true);
create policy "all"  on {table} for all using (true) with check (true);
```
Role enforcement is client-side only.

## 15. Page Patterns

### Dashboard Page
- Calls a `useDashboardKpis()` hook that runs `Promise.all()` of multiple count/aggregate queries
- Renders a grid of `<KpiCard />` components: `grid grid-cols-1 sm:grid-cols-2 lg:grid-cols-4 gap-4`
- Below KPIs: Recharts charts + recent activity tables 
- Shows loading/error states via `<LoadingSpinner />` and `<ErrorMessage />`

### List Page
```jsx
export default function EntityList() {
  const [filters, setFilters] = useState({})
  const { data, isLoading, error } = useEntities(filters)
  const [showForm, setShowForm] = useState(false)

  if (isLoading) return <LoadingSpinner />
  if (error) return <ErrorMessage message={error.message} />

  return (
    <div className="space-y-6">
      <PageHeader title="..." description="..." actions={<button onClick={() => setShowForm(true)}>+ New</button>} />

      {/* Filter bar */}
      <div className="flex flex-wrap gap-3">
        <select className="rounded-lg border-gray-300 text-sm" onChange={...}> ... </select>
      </div>

      {/* Results */}
      {data?.length === 0 ? (
        <EmptyState icon={Icon} title="No items" description="..." />
      ) : (
        <div className="bg-white rounded-xl border border-gray-200 overflow-hidden">
          <table className="w-full text-sm">
            <thead className="bg-gray-50 text-gray-500 text-xs uppercase">
              <tr><th className="px-4 py-3 text-left">...</th></tr>
            </thead>
            <tbody className="divide-y divide-gray-100">
              {data?.map(item => <tr key={item.id}>...</tr>)}
            </tbody>
          </table>
        </div>
      )}

      {/* Modal form */}
      {showForm && <FormModal onClose={() => setShowForm(false)} />}
    </div>
  )
}
```

## 19. Naming Conventions Summary

| Element | Convention | Example |
|---|---|---|
| Component files | PascalCase.jsx | `KpiCard.jsx`, `GrowCycleDetail.jsx` |
| Hook files | camelCase.js | `useSupabase.js`, `useManagement.js` |
| Lib/service files | camelCase.js | `supabase.js`, `d365Api.js` |
| Context files | PascalCase.jsx | `AuthContext.jsx` |
| DB tables | snake_case, plural | `grow_cycles`, `harvest_schedules` |
| DB columns | snake_case | `full_name`, `area_acres`, `is_active` |
| DB FKs | `{table_singular}_id` | `field_id`, `crop_id` |
| React Query keys | kebab-case strings | `['grow-cycles']`, `['dashboard-kpis']` |
| Query hooks | `use{Entity}` | `useGrowCycles`, `useDashboardKpis` |
| Mutation hooks | `useCreate/Update/Delete{Entity}` | `useCreateGrowCycle` |
| Route paths | kebab-case | `/grow-cycles`, `/harvest-scheduling` |
| CSS tokens | kebab-case with scope | `--color-primary-600`, `--color-harvest-500` |
| Env vars (frontend) | `VITE_` prefix, SCREAMING_SNAKE | `VITE_SUPABASE_URL` |
| Env vars (edge fn) | SCREAMING_SNAKE | `D365_TENANT_ID` |

## 20. Quick-Start Checklist for a New Project

1. `npm create vite@latest {project-name} -- --template react`
2. `cd {project-name} && npm install`
3. Install dependencies:
   ```
   npm install @supabase/supabase-js @tanstack/react-query react-router-dom lucide-react recharts date-fns
   npm install -D tailwindcss @tailwindcss/vite
   ```
4. Configure `vite.config.js` (see §6)
5. Replace `src/index.css` with Tailwind v4 imports + `@theme` tokens (see §4)
6. Create `src/main.jsx` provider hierarchy (see §5)
7. Create `.env` with `VITE_SUPABASE_URL` and `VITE_SUPABASE_ANON_KEY`
8. Create Supabase project, run migration SQL files
9. Build out: `lib/supabase.js` → `contexts/AuthContext.jsx` → `components/` → `hooks/` → `pages/` → `App.jsx`
10. Create `swa-cli.config.json` for Azure SWA deployment
11. (If D365) Create `supabase/functions/d365-proxy/index.ts`, set secrets, deploy
12. Seed demo data and verify
